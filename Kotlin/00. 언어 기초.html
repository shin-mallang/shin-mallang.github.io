0. 언어 기초


1. 주석

  • 한 줄 주석: //
  • 여러 줄 주석: /* 여러 줄 주석 */
  • KDoc 여러 줄 주석: /** KDoc 여러 줄 주석 */

자바와 달리 코틀린에서는 주석 내 주석 사용이 가능하다.
예를 들면 다음과 같다.

/*
여러 줄 주석
/* 주석 내 주석 */
*/


2. 변수 정의

  • val: 불변(immutable) 변수를 정의한다.
  • var: 가변(mutable) 변수를 정의한다.

자바와 달리 코틀린에서 대입(assignment)는 statement 이다.
즉 아무런 값을 반환하지 않는다.

Statement 와 Expression
  • Statement
    • 실행 시 프로그램의 상태전이를 수행하는 언어의 구성요소
    • 값을 반환하지 않는다.
  • Expression
    • 실행시 값으로 계산되는 언어의 구성요소
    • Expression의 실행 결과는 항상 '값'을 반환한다.


3. 문자

코틀린에서는 문자열 템플릿(string template) 기능을 제공한다.
이를 사용하면 문자열 내부에 Expression 을 추가할 수 있다.

fun main() {
	val name = readLine()
	println("Hello, $name, Today is ${Date()}")
}

3.1 문자열 접근

문자열 내 특정 순서의 문자에 접근할 때에는 배열과 같이 접근할 수 있다.

fun main() {
	val name = "mallang"
	println(name[0])  // m
	println(name[1])  // a
	println(name[2])  // l
}


4. 배열

배열은 다음과 같이 정의할 수 있다.

val a = emptyArray<String>()  // 원소 1개
val b = arrayOf("1", "2")     // 원소 2개
val c = arrayOf(1, 2, 3)      // 원소 3개

val size = 10
val d = Array(size) { it }    // 0, 1, 2, ... , 9

사용은 다음과 같다.

val array = arrayOf(1, 2, 3, 4)
println(array.size)  // 4
println(array[0])  // 1
println(array[2])  // 3

// 원소 변경
array[0] = 10
println(array[0])  // 10

참고로 자바의 배열은 공변한 성질을 가지지만, 코틀린의 배열은 무공변한 성질을 갖는다.
즉 자바에서는 상위 타입의 배열에 하위 타입의 배열을 대입할 수 있으나,
코틀린에서는 불가능하다.



5. 함수

코틀린의 함수 구조는 다음과 같다.

import kotlin.math.PI

fun circleArea(radius: Double): Double {
	return PI * radius * radius
}


리턴 타입이 없는 경우(자바의 경우 void) 다음과 같이 사용한다.

fun sample(param: Int): Unit {
	println(param)
}

// 반환 타입이 Unit 인 경우에는 생략 가능하다.
fun sample(param: Int) {
	println(param)
}

자바와 달리 코틀린에서는 파라미터가 불변 타입이다.
즉 다음과 같은 코드는 컴파일 오류가 발생한다.

fun sample(param: Int) {
	param = 10
	println(param)
}

코틀린에서는 expression-body 함수를 지원한다.
어떤 함수가 단일 식(expression)으로만 구현될 수 있다면 다음과 같은 형태로 함수를 작성해도 된다.

fun circleArea(radius: Double): Double = PI * radius * radius

그리고 이러한 경우에는 반환 타입 생략이 가능하다.

fun circleArea(radius: Double) = PI * radius * radius

다음과 같이도 사용 가능하다.

fun circleArea(radius: Double) = {
	PI * radius * radius
}

5.1 named argument

코틀린에서는 함수에 인자를 전달할 때, 이름을 기반으로 인자를 전달하는 방식도 제공한다.

fun sample(a: Int, b: Int) {  
    println("a = $a, b = $b")  
}  
  
fun main() {  
    sample(1, 2)  // a = 1, b = 2
    sample(b = 1, a = 2)  // a = 2, b = 1
}

5.2 default parameter

코틀린에서는 함수의 파라미터에 기본 값을 지정해줄 수 있다.

fun sample(a: Int = 10, b: Int = 5) {  
    println("a = $a, b = $b")  
}  
  
fun main() {  
    sample(1, 2)  // a = 1, b = 2
    sample(b = 1, a = 2)  // a = 2, b = 1
    sample(b = 1)  // a = 10, b = 1
}

5.3 vararg

인자의 개수가 정해지지 않은 경우, 즉 가변인자에는 vararg를 사용할 수 있다.
(자바에서는 ... 를 통해 사용 가능하다.)

fun sample(vararg params: Int) {
	// 내부에서 params 의 타입은 array 임
	println(params.contentToString())
}

fun main() {
	sample(1, 2, 3)
}

배열을 가변 인자로 넘기기 위해서는 스프레드(spread) 연산자인 *를 사용할 수 있다.

fun sample(vararg params: Int) {
	// 내부에서 params 의 타입은 array 임
	println(params.contentToString())
}

fun main() {
	val a = intArrayOf(1, 2, 3)
	// sample(a) // 컴파일 오류 -> passing IntArray instead of Int
	sample(*a)
}
